# Causal Impact Playground

施策前後比較・A/Bテスト・Difference-in-Differences (DID) の違いを、合成データをいじりながら可視化する小型アプリ。因果推論と実験設計の考え方を直感的に示す。

> **注意**: これは合成データの実験であり、実在の因果効果を示すものではありません。

## 目的

- 単純前後差がトレンドによってバイアスされることを体感する
- 対照群を用いた DID が平行トレンド仮定の下で真の効果に近づくことを確認する
- 平行トレンドが崩れると DID もバイアスされることを可視化する
- 推定結果の過剰解釈を防ぐ注意文を常に併示する

## 主要機能

- 処置群と対照群の合成パネルデータ生成（シード固定で再現性あり）
- 施策効果・ノイズ・トレンド差・群レベル差をスライダーで操作
- 群別トレンドグラフ（施策導入タイミングを縦線表示）
- 単純前後差と DID の推定値を 95% 信頼区間付きで比較
- 真の効果と推定値のバイアス・包含判定を表で表示
- 事前期間の平行トレンド診断（群別傾きとその差）
- 平行トレンド崩壊時の警告と過剰解釈防止の注意文

## 使用技術

- Python 3.11+
- Streamlit（UI）
- NumPy / SciPy / statsmodels（データ生成・OLS推定・信頼区間）
- pandas / matplotlib（集計・可視化）
- pytest（推定ロジックのテスト）

## データの出所

すべて合成データ。`src/data.py` のモデルに従い乱数で生成する:

```
y = baseline + trend*t + group_offset*treat + trend_diff*treat*t
    + true_effect*treat*post + N(0, noise_sd)
```

- `treat`: 処置群=1 / 対照群=0
- `post`: 施策後期間=1 / 事前=0
- `trend_diff != 0` のとき平行トレンド仮定が崩れる
- 真の効果 `true_effect` は処置群の事後期間にのみ加算される

実データ・実在の施策・機密情報は一切使用しない。APIキーも不要。

## ローカル実行手順

```bash
cd causal-impact-playground

# 仮想環境の作成と依存インストール（uv を推奨）
uv venv
uv pip install -e ".[dev]"
# または: python -m venv .venv && source .venv/bin/activate && pip install -e ".[dev]"

# アプリ起動
streamlit run app/streamlit_app.py
```

ブラウザが開くので、左のサイドバーでパラメータを変えると推定値がリアルタイムに更新される。

### テスト

```bash
pytest
```

`src/` のデータ生成と推定関数に対する最低限のテストが走る。

## スクリーンショット

スクリーンショットは `assets/` に配置する。

- `assets/trend.png`: 群別トレンドグラフ
- `assets/estimates.png`: 推定値と真の効果の比較

## 制限事項

- 合成データのみ。実データの因果効果を主張するものではない。
- DID の平行トレンド検査は事前期間の傾き比較による簡易診断であり、厳密な統計検定ではない。
- ノイズ・サンプルサイズによっては信頼区間が真値を外れることがある（シミュレーションの性質）。
- 認証・DB・本番運用機能は持たない（MVP）。
- モバイル幅でも表示は崩れないが、横長グラフはスクロール前提。
